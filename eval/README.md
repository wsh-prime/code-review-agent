# Eval

## Responsibility

This directory will contain benchmark data, metric scripts, and ablation experiments for evaluating the Code Review Agent Harness.

## Current Status

Phase 0 only includes this README as a placeholder. No benchmark or metric scripts exist yet.

## Files

| File | Responsibility |
|---|---|
| `README.md` | Describes the planned evaluation area. |
| `benchmark/` | Planned: store curated real PR cases and injected-bug cases. |
| `run_benchmark.py` | Planned: run the review agent across benchmark cases. |
| `metrics.py` | Planned: calculate precision, recall, F1, and false positive rate. |
| `ablation.py` | Planned: compare full agent against variants without RepoMap, Hygiene, or chunking. |

## Inputs

- Review reports generated by the agent.
- Curated benchmark cases.
- Human review labels or injected-bug metadata.

## Outputs

- Metric summaries.
- Ablation results.
- Future evaluation reports for README and resume storytelling.

## Not Responsible For

- Running the core review workflow.
- Formatting user-facing review reports.
- Storing production configuration or secrets.

## Next Changes

- Wait until MVP review reports are stable before adding benchmark scripts.
- Define a compact fixture format for real PR and injected-bug cases.
